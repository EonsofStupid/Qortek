# Fork / Deliberate / Commit / Foldback

Qortek models branch-worthy context as a first-class workflow.

```text
Seed discussion
  -> fork opens
  -> branch deliberates
  -> branch reaches commit condition
  -> foldback collapses the branch into seed context
```

## Why this exists

Vague operator context often contains multiple hidden topics. Instead of derailing the seed discussion, Qortek can open a fork, resolve the branch, and fold the result back as durable context.

## User-facing terms

- Fork: branch topic created from ambiguity or a side mission.
- Deliberate: gather context, reason, retrieve, or build a plan.
- Commit: mark the branch result as useful.
- Foldback: collapse the result into the seed conversation and memory graph.

## Internal term

The foldback step is catamorphic: a branching structure is reduced into a durable summary, memory, decision, or plan.
