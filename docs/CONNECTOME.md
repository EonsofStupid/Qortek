# Connectome

Qortek's connectome is the durable relationship layer that lets RROs, memories, tools, forks, vector hits, and estate nodes become one navigable reasoning substrate.

The connectome is inspired by multi-model database ideas, but Qortek keeps its own public model:

```text
BrainStore = durable truth
Graph relationships = meaning/pathways
VectorRecall = recall heart
Mesh = estate/node reachability
Preflight = route selection
RRD = orchestration
```

## Core idea

A vector hit should not be an isolated chunk. It should resolve back into the connectome:

```text
vector hit
  -> source RRO
  -> memory declaration
  -> graph edge
  -> workspace boundary
  -> fork or seed chat
  -> evidence / tool trace
```

## Growing as the operator shares

The connectome expands whenever the operator shares context:

- conversations
- files
- repos
- docs
- tools
- screenshots
- estate nodes
- decisions
- project plans

Each new input becomes a candidate RRO, graph relationship, vector artifact, and audit event.

## Heart/pathway model

Qortex is the recall heart. It pumps relevant vector hits through established connectome pathways.

The connectome supplies:

- relationships
- provenance
- trust boundaries
- workspace scope
- fork lineage
- memory lifecycle

Qortex supplies:

- dense recall
- sparse recall
- hybrid retrieval
- ANN performance
- rerank-ready candidates

Neither layer replaces the other.
